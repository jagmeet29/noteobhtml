Comprehensive Guide to Verilog HDL: Fundamentals, Data Types, and Net Strengths

1. Verilog Language Fundamentals

Verilog is a Hardware Description Language (HDL) that allows designers to model digital systems at various levels of abstraction. Understanding its basic syntax and structural elements is crucial.

1.1 Case Sensitivity

Verilog is case-sensitive. This means var_a and var_A are treated as different identifiers. All statements typically terminate with a semicolon (;).

1.2 Comments

Verilog supports two types of comments:

  • Single-line comments: Start with // and extend to the end of the line.
  • Multi-line comments: Start with /* and end with */. These cannot be nested. However, single-line comments can be nested within multi-line comments.
// This is a single line comment.

integer a;   // Creates an int variable called a.

/*
This is a
multiple-line or
block comment
*/;

/* This is /* an invalid nested block comment */ */ // This line would cause an error
/* However, // this one is okay */ // This is valid

1.3 Whitespace

Whitespace (spaces, tabs, newlines, formfeeds) is generally ignored by Verilog, except when it separates tokens or within strings. Whitespace is used to improve code readability through indentation.

module dut;              // 'module' is a keyword, 'dut' is an identifier.
reg [8*6:1] name = "Hello!";   // Spaces at the beginning are ignored.

In strings, however, whitespace is preserved:

   // There is no space in the beginning of this line,
   // but there's a space in the string.
   reg [8*6:1] addr = "Earth "; // The space after "Earth" is part of the string.
endmodule;

1.4 Operators

Operators perform operations on one or more operands.

  • Unary operators: Appear to the left of their single operand (e.g., ~y).
  • Binary operators: Appear between their two operands (e.g., y | z).
  • Ternary/Conditional operators: Have two operators separating three operands (e.g., (y > 5) ? w : z). If (y > 5) is true, x gets w; otherwise, x gets z.
x = ~y;                // ~ is a unary operator, and y is the operand.
x = y | z;             // | is a binary operator, where y and z are its operands.
x = (y > 5) ? w : z;   // ?: is a ternary operator.

1.5 Identifiers

Identifiers are names given to variables, modules, ports, etc., for later reference. They consist of alphanumeric characters (a-z, A-Z, 0-9), underscores (_), or dollar signs ($). They are case-sensitive.
Rules:

  • Cannot start with a digit or a dollar sign.
integer var_a;        // Valid: alphanumeric and underscore.
integer $var_a;       // Invalid: starts with $.
integer v$ar_a;       // Valid: contains $.
integer 2var;         // Invalid: starts with a digit.
integer var23_g;      // Valid: alphanumeric and underscore.
integer 23;           // Invalid: only numbers, treated as a literal.

1.6 Keywords

Keywords are special, reserved identifiers that define language constructs. They are always in lowercase. Examples include module, endmodule, wire, reg, always, initial, assign, etc.

1.7 Numbers and Radix

Numbers in Verilog can be represented in decimal, binary, octal, or hexadecimal. By default, numbers are treated as decimal.

Sized Numbers

Sized numbers explicitly define their bit width and base format:
[size]'[base_format][number]

  • size: Number of bits (decimal only).
  • base_format: d or D (decimal), h or H (hexadecimal), o or O (octal), b or B (binary).
  • number: Digits corresponding to the base_format. For hexadecimal, A-F (case-insensitive) are allowed.
3'b010;     // 3-bit binary number with value 2.
3'd2;       // 3-bit decimal number with value 2.
8'h70;      // 8-bit hexadecimal number (0x70) representing decimal 112.
9'h1FA;     // 9-bit hexadecimal number (0x1FA) representing decimal 506.

4'hA = 4'd10 = 4'b1010 = 4'o12; // Decimal 10 in different formats.
8'd234 = 8'D234;                 // 'd' or 'D' are legal.
32'hFACE_47B2;                  // Underscore (_) improves readability.

Numbers without a base_format are decimal by default. Numbers without a size default to a simulator/machine-dependent bit width (typically 32 bits for integer).

integer a = 5423;       // 'a' gets decimal value 5423.
integer a = 'h1AD7;     // 'a' (32-bit integer) gets 32'h0000_1AD7.

Negative Numbers

Negative numbers are specified by placing a minus sign (-) before the size. They are stored in 2's complement form.

-6'd3;            // 6-bit negative number, 2's complement of 3.
-6'sd9;           // Signed 6-bit negative 9.
8'd-4;            // Illegal: minus sign between size and base format.

Verilog Padding Rules

When a literal's size is larger than its number's specified bits:

  • If the MSB (leftmost bit) is x or z, the remaining bits are padded with x or z.
  • If the MSB is 0 or 1, the remaining bits are padded with zeros.

1.8 Strings

A sequence of characters enclosed in double quotes (" ") is a string. Each character takes 1 byte (8 bits). Strings cannot span multiple lines.

  • If a variable's width is smaller than the string, the string is truncated from the left.
  • If a variable's width is larger, zeros (spaces for display) are padded to the left.
"Hello World";        // 11 characters -> 11 bytes.
"x + z";               // 5 characters.

// Illegal: "How are you feeling today ?"; // String split into multiple lines.

module testbench;
  reg [8*11:1] str1; // 11 bytes wide
  reg [8*5:1]  str2; // 5 bytes wide
  reg [8*20:1] str3; // 20 bytes wide

  initial begin
    str1 = "Hello World";
    str2 = "Hello World";
    str3 = "Hello World";

    $display ("str1 = %s", str1); // Full string
    $display ("str2 = %s", str2); // Truncated: "World"
    $display ("str3 = %s", str3); // Padded: "         Hello World"
  end
endmodule

2. Verilog Value System

Verilog uses a 4-state logic system to model hardware behavior, representing voltage levels and unknown states.
| Value | Description |
| :---- | :------------------------------------------- |
| | Logic zero, 0V, or a false condition |
| | Logic one, Vdd, or a true condition |
| | An unknown logic value (could be 0 or 1) |
| | A high-impedance state (unconnected wire) |

X (x) means the value is unknown (not necessarily "don't care" as in boolean logic). Z (z) represents a high-impedance state, similar to an unconnected wire.

3. Verilog Data Type Categories: Nets and Variables

Verilog data types fall into two fundamental categories that model different hardware elements and behave differently in terms of how they retain values and are assigned.

  • Net Types: Represent physical connections between digital circuits. They must be continuously driven and do not store values (except trireg).
  • Variable/Register Types: Represent data storage elements that retain values until overridden. They do not require continuous driving.

Nets

Nets are analogous to physical wires connecting hardware components (e.g., logic gates, flip-flops). They do not store values themselves; their value is continuously determined by their drivers. The most common net type is wire.

Variables (Registers)

Variables (or register types) are abstractions of data storage elements (e.g., flip-flops). They can hold a value until a new value is assigned. They are used for behavioral modeling and can appear on the left-hand side of procedural assignments. A reg type variable does not necessarily synthesize into a hardware register; it can also be used to specify combinational logic.

4. Detailed Verilog Net Types

Net data types model connections in structural descriptions and are continuously driven.

4.1 Common Net Types

  • wire: Basic interconnecting wire for single-driver connections. When multiple drivers conflict, results in x (unknown).
  • tri: Identical to wire but conventionally used for multiple-driver nets. Behaves like wire with multiple drivers.
  • wor / trior: Wired OR configurations. If any driver outputs 1, the net becomes 1.
  • wand / triand: Wired AND configurations. If any driver outputs 0, the net becomes 0.
  • tri0: Models nets with resistive pulldown devices. Defaults to 0 if undriven or z.
  • tri1: Models nets with resistive pullup devices. Defaults to 1 if undriven or z.
  • supply0: Power supply connection driven to a constant logic 0. Has the highest signal strength for 0.
  • supply1: Power supply connection driven to a constant logic 1. Has the highest signal strength for 1.
  • trireg: Stores values when tri-stated, modeling capacitive storage. Can retain its last driven value when all drivers are in high-impedance (z).
  • uwire: Unresolved or unidriver wire. Allows only a single driver; multiple drivers result in a compile-time error.

4.2 Net Behavior Examples

When multiple drivers conflict:

  • Standard wire/tri: Results in x (unknown) when driven with conflicting values (e.g., 0 and 1).
  • wor/trior: Performs logical OR. Becomes 1 if any driver outputs 1.
  • wand/triand: Performs logical AND. Becomes 0 if any driver outputs 0.
  • tri0: Equivalent to wire with a continuous resistive pulldown. Becomes 0 if undriven or z.
  • tri1: Equivalent to wire with a continuous resistive pullup. Becomes 1 if undriven or z.

Simulation Example for Wired Logic Nets and Tri-state Nets

  wor  		wor_net;
  wand 		wand_net;

  wire      normal_net;

  reg 		driver_1;
  reg 		driver_2;
  reg [3:0] values; // For Z, X, 1, 0

  assign wor_net = driver_1;
  assign wor_net = driver_2;
  assign trior_net = driver_1;
  assign trior_net = driver_2;
  assign wand_net = driver_1;
  assign wand_net = driver_2;
  assign triand_net = driver_1;
  assign triand_net = driver_2;
  assign tri0_net = driver_1;
  assign tri0_net = driver_2;

  assign tri1_net = driver_2;

5. Detailed Verilog Register/Variable Types

Register types represent data storage elements and retain their last assigned value. They are used in procedural blocks (always, initial).

5.1 reg Data Type

The most widely used register type.

  • Default value: 'x' (unknown).
  • Default size: 1-bit (can be explicitly specified).
  • Declaration format: 
    reg x, y;          // Single-bit register variables
reg [15:0] bus;    // A 16-bit bus (vector)
  • Arithmetic treatment: Unsigned numbers in arithmetic expressions.
  • Primary use: Modeling sequential hardware elements (counters, shift registers) and combinational logic within procedural blocks.

Example - Synchronous Counter:

module simple_counter (clk, rst, count);
    input clk, rst;
    output [31:0] count;
    reg [31:0] count;
    always @(posedge clk) begin // Only sensitive to positive edge of clk
        if (rst)
            count = 32'b0;
        else
            count = count + 1;
    end
endmodule

Example - Asynchronous Counter:

module simple_counter (clk, rst, count);
    input clk, rst;
    output [31:0] count;
    reg [31:0] count;
    always @(posedge clk or posedge rst) begin // Sensitive to both clk and rst
        if (rst)
            count = 32'b0;
        else
            count = count + 1;
    end
endmodule

5.2 integer Data Type

Offers enhanced functionality for general-purpose operations.

  • Default size: 32 bits.
  • Arithmetic treatment: 2's complement signed integer.
  • Size optimization: Synthesis tools determine optimal size.
  • Primary use: Loop counting, general-purpose register operations, mathematical operations (more suitable than reg for signed arithmetic).

Example:

wire [15:0] X, Y;
integer C; // Automatically 32-bit signed
wire Z;
// Z = X + Y; // This would cause a width mismatch error if Z is 1-bit
// If Z is also a wire, its width needs to be sufficient
reg [16:0] sum_result; // X+Y can result in 17 bits (16 data + 1 carry)
always @(*) begin
    sum_result = X + Y;
end

5.3 real Data Type

Handles floating-point numbers.

  • Purpose: Store floating-point numbers.
  • Conversion behavior: When assigned to an integer, rounds to the nearest integer.
  • Format: Standard floating-point notation.

Example:

real e, pi;
integer x;
initial begin
    e = 2.718;
    pi = 3.14159e-2; // 0.0314159
    x = pi;  // Gets value 0 (rounded from 0.0314159)
    $display("e = %0.3f, pi = %0.5f, x = %0d", e, pi, x);
    pi = 3.14159;
    x = pi;  // Gets value 3 (rounded from 3.14159)
    $display("pi = %0.5f, x = %0d", pi, x);
end

5.4 time Data Type

Manages simulation timing.

  • Purpose: Store simulation time values.
  • System function: $time returns current simulation time.
  • Usage: Primarily for simulation and debugging (not synthesizable).
  • Default size: 64-bit unsigned.

Example:

time curr_time;
initial begin
    #20; // Advance simulation time by 20 units
    curr_time = $time;
    $display("Current simulation time: %0t", curr_time);
end

5.5 Important Register Characteristics

  • Register data types must be used when a signal appears on the left-hand side of procedural assignments (always, initial).
  • They can hold values until the next assignment and don't require continuous driving.
  • A reg declaration does not necessarily map to a hardware register during synthesis; it can describe combinational logic.

6. Scalar vs. Vector Declarations

Verilog can represent individual bits (scalars) or groups of bits (vectors).

6.1 Scalar Declaration

A net or reg declaration without a range specification is a 1-bit wide scalar.

wire 	    o_nor;           // Single-bit scalar net
reg         parity;          // Single-bit scalar variable

6.2 Vector Declaration

If a range is specified, the net or reg becomes a multibit entity known as a vector.

wire [7:0]  o_flop;          // 8-bit vector net
reg  [31:0] addr;            // 32-bit vector variable

The range specifies the Most Significant Bit (MSB) as the left-hand value and the Least Significant Bit (LSB) as the right-hand value (e.g., [MSB:LSB] or [LSB:MSB]). MSB and LSB must be constant expressions.

6.3 Accessing Vector Elements

Bit-Select

Individual bits within a vector can be accessed using a bit-select.

reg [7:0]      addr;         // 8-bit reg variable [7, 6, 5, 4, 3, 2, 1, 0]

addr [0] = 1;                // Assign 1 to bit 0 of addr
addr [3] = 0;                // Assign 0 to bit 3 of addr
addr [8] = 1;                // Illegal: bit 8 is out of bounds (returns x if read)

Part-Select

A range of contiguous bits can be selected as a part-select.

  • Constant Part-Select: Fixed start and end bits. 
    reg [31:0] addr;
addr [23:16] = 8'h23; // bits 23 to 16 replaced by 0x23
  • Indexed Part-Select: Variable starting bit, constant width. Useful in loops.
    • [<start_bit> +: <width>]: Increments from start_bit.
    • [<start_bit> -: <width>]: Decrements from start_bit.
module des;
  reg [31:0]  data;
  int         i;

  initial begin
    data = 32'hFACE_CAFE; // FA CE CA FE
    for (i = 0; i < 4; i++) begin
      // data[8*0 +: 8] => data[7:0]
      // data[8*1 +: 8] => data[15:8]
      // data[8*2 +: 8] => data[23:16]
      // data[8*3 +: 8] => data[31:24]
      $display ("data[8*%0d +: 8] = 0x%0h", i, data[8*i +: 8]);
    end
  end
endmodule

Simulation Log Snippet:

data[8*0 +: 8] = 0xfe
data[8*1 +: 8] = 0xca
data[8*2 +: 8] = 0xce
data[8*3 +: 8] = 0xfa

Note: data[0:9] for reg [15:0] data is illegal. The order of bits in a part-select must match the declaration order ([MSB:LSB] or [LSB:MSB]).

7. Arrays and Memories in Verilog

Verilog supports arrays of nets and variables, which can be scalar or vector, and multi-dimensional. Arrays are allowed for reg, wire, integer, and real data types.

7.1 Array Declaration and Access

An array is declared by specifying an address range after the identifier name.

reg        y1 [11:0];        // Scalar reg array: 12 elements, each 1-bit wide
wire [0:7] y2 [3:0];         // Vector net array: 4 elements, each 8-bit wide
reg  [7:0] y3 [0:1][0:3];    // 2D vector reg array: 2 rows, 4 columns, each 8-bit wide

Individual elements are accessed by indexing each dimension. Assigning to the whole array at once (e.g., y1 = 0;) is illegal.

y2[0] = 8'ha2;           // Assign 0xa2 to element at index 0 of y2
y3[1][2] = 8'hdd;     // Assign 0xdd to element at rows=1, cols=2 of y3

7.2 Example

module des ();
  reg [7:0]  mem1; 							// reg vector 8-bit wide
  reg [7:0]  mem2 [0:3]; 				// 8-bit wide vector array with depth=4
  reg [15:0] mem3 [0:3][0:1]; 	// 16-bit wide vector 2D array with rows=4,cols=2

  initial begin
    int i, j;

    mem1 = 8'ha9;
    $display ("mem1 = 0x%0h", mem1);

    mem2[0] = 8'haa;
    mem2[1] = 8'hbb;
    mem2[2] = 8'hcc;
    mem2[3] = 8'hdd;
    for(i = 0; i < 4; i = i+1) begin
      $display("mem2[%0d] = 0x%0h", i, mem2[i]);
    end

    for(i = 0; i < 4; i += 1) begin
      for(j = 0; j < 2; j += 1) begin
        mem3[i][j] = i + j;
        $display("mem3[%0d][%0d] = 0x%0h", i, j, mem3[i][j]);
      end
    end
  end
endmodule

7.3 Memories

Memories in Verilog are typically modeled using one-dimensional arrays of reg type. Each element in the array represents a "word" and is referenced by an index.

Verilog Vectors vs. Memory Arrays

  • A Verilog vector (e.g., reg [15:0] my_reg;) declares a single multi-bit register. This directly maps to a single block of flip-flops of that width.
  • A memory array (e.g., reg [15:0] my_memory [0:3];) declares an array of registers. This maps to multiple independent multi-bit registers, each addressable by an index.

Single Register Example:

module des (input clk, input rstn, input wr, input sel, input [15:0] wdata, output [15:0] rdata);
	reg [15:0] register; // Single 16-bit register
	always @ (posedge clk) begin
    if (!rstn) register <= 0;
    else begin
    	if (sel & wr) register <= wdata;
    	else register <= register; // Retain current value
    end
	end
	assign rdata = (sel & ~wr) ? register : 0;
endmodule

Hardware Schematic (Conceptual): A single 16-bit flip-flop with control logic.

Register Array Example (Memory):

module des (input clk, input rstn, input [1:0] addr, input wr, input sel, input [15:0] wdata, output [15:0] rdata);
reg [15:0] register [0:3]; // Array of 4 x 16-bit registers
integer i;
always @ (posedge clk) begin
    if (!rstn) begin
        for (i = 0; i < 4; i = i+1) begin
            register[i] <= 0; // Initialize all elements
        end
    end else begin
        if (sel & wr)
            register[addr] <= wdata; // Write to addressed element
        else
            register[addr] <= register[addr]; // Retain addressed element's value
    end
end
assign rdata = (sel & ~wr) ? register[addr] : 0; // Read from addressed element
endmodule

Hardware Schematic (Conceptual): Four independent 16-bit flip-flops, with a decoder to select which one is accessed based on addr.

What is the difference between Array and Vector?
Answer

A vector refers to a single multi-bit net or variable (e.g., wire [7:0] data_bus;). It represents a single piece of data composed of multiple bits.

An array is a collection of elements of the same data type. Each element in the array can be a scalar or a vector. For example, reg [7:0] memory [0:255]; declares an array of 256 elements, where each element is an 8-bit vector.

Key Distinction: A vector is a single item of multiple bits, while an array is a collection of items (which themselves can be single bits or vectors). You access bits of a vector using bit-select (vector[idx]) or part-select (vector[high:low]). You access elements of an array using an array index (array[idx]) and then potentially a bit-select on that element (array[idx][bit_idx]).

8. Net Driving Strengths in Verilog

In Verilog, strength refers to the relative power or capability of a driver to influence the value of a net. This is critical for resolving conflicts when multiple drivers attempt to control the same net.

8.1 Types of Strengths

Charge Strength

Used with trireg nets to model charge storage (e.g., in capacitors). It indicates the relative size of the capacitance: small, medium, or large. This strength determines how quickly a charge can decay. Default for trireg is medium.

trireg                          a_net;    // strength medium by default
trireg   (medium) #(0, 0, 100)  cap1;     // strength medium, charge decay time of 100 time units
trireg   (large)  [3:0]         cap2;     // strength large, no decay time

Drive Strength

Refers to a driver's ability to influence a net's value. Higher strength drivers override lower strength drivers. If multiple drivers of the same strength have conflicting values, the net resolves to x (unknown).

Drive strengths are specified in assign statements or gate instantiations using (strength1, strength0):
assign (strength1, strength0) net = expression;

  • strength1: Strength when driving logic 1.
  • strength0: Strength when driving logic 0.
    Default drive strength is (strong1, strong0).

8.2 Specific Drive Strengths (Hierarchical Order from Strongest to Weakest)

  1. supply1 / supply0: Represents power supply connections. These are the strongest drivers, ensuring a constant 1 or 0.
  2. strong1 / strong0: Indicates active, strong driving capability. Overrides weaker strengths.
  3. pull1 / pull0: Represents resistive pull-up/pull-down devices. Drives 1 or 0 when no stronger drivers are active.
  4. weak1 / weak0: Indicates weak driving capability. Easily overridden by stronger drivers.
  5. highz1 / highz0: Indicates a high-impedance state (z). This is a special case:
    • highz1 implies the driver pulls the net to z when trying to drive 1.
    • highz0 implies the driver pulls the net to z when trying to drive 0.
    • Illegal constructs: (highz1, highz0) and (highz0, highz1) are not allowed. A driver cannot simultaneously drive a 1 with high-Z and a 0 with high-Z.

Example assign with strengths:

wire out;
assign (strong1, weak0) out = a & b; // Drives 'out' with strong1 when (a&b) is 1, and weak0 when (a&b) is 0.

8.3 Drive Strength Rules and Constraints

  • The strength specifications (highz1, highz0) and (highz0, highz1) are illegal.
  • If drive strength is not specified, it defaults to (strong1, strong0).

8.4 Example and Simulation

This example demonstrates how different drive strengths influence a net's value based on conflicting drivers.

module tb;
    // Declare nets with different strengths
    reg a, b, c, d;

    wire strong0_net, strong1_net;
    wire pull0_net, pull1_net;
    wire weak0_net, weak1_net;
    wire highz0_net, highz1_net;

    // Assign strong drivers
    assign (strong1, weak0) strong1_net = a; // Drives high strongly if 'a' is 1, low weakly if 'a' is 0
    assign (weak1, strong0) strong0_net = b; // Drives high weakly if 'b' is 1, low strongly if 'b' is 0

    // Assign pull drivers
    assign (pull1, pull0) pull1_net = c; // Pulls up to 1 unless driven low by a stronger driver
    assign (pull0, pull1) pull0_net = d; // Pulls down to 0 unless driven high by a stronger driver

    // Assign weak drivers
    assign (weak1, weak0) weak1_net = a; // Weakly drives high if 'a' is 1, low weakly if 'a' is 0
    assign (weak0, weak1) weak0_net = b; // Weakly drives high if 'b' is 1, low weakly if 'b' is 0

    // Assign high impedance drivers (illustrative, for high-Z output capability)
    // Note: highzX is usually for outputs of gates/modules, not direct assign.
    // The example uses pullX for the 0/1 side of highz for illustrative purposes.
    assign (highz1, pull0) highz1_net = a; // If 'a' is 1, net goes to Z, else 'pull0'
    assign (highz0, pull1) highz0_net = b; // If 'b' is 0, net goes to Z, else 'pull1'

    initial begin
      reg [1:0] values = {1'b1, 1'b0}; // For random '1' or '0' inputs

      repeat (10) begin
        integer idx;
        #10;
        idx = $random % 2; a = values[idx];
        idx = $random % 2; b = values[idx];
        idx = $random % 2; c = values[idx];
        idx = $random % 2; d = values[idx];
      end
    end

    initial
      $monitor("[%0t] a=%0b b=%0b c=%0b d=%0b strong1=%0b strong0=%0b pull1=%0b pull0=%0b weak1=%0b weak0=%0b highz1=%0b highz0=%0b",
               $time, a, b, c, d, strong1_net, strong0_net, pull1_net, pull0_net, weak1_net, weak0_net, highz1_net, highz0_net);

endmodule
